# 轻量哨兵 — 核心规则

> 零依赖版本。只扫描、提取、存档，不通知、不同步。
> 状态文件：`output/sentinel-status.md`

---

## 配置

| 变量 | 默认值 | 说明 |
|---|---|---|
| `SENTINEL_NAME` | 轻量哨兵 | 本 skill 的显示名称。可自定义。 |
| `WORKSPACE_DIR` | `./output/` | 工作目录，相对于 skill 目录。所有输出文件放这里。 |

> **使用方式**：所有下文中的 `{SENTINEL_NAME}` 均引用上表的值。
> CC / Codex 执行时读取此变量，替换到输出文本、文件名标记中。

---

## 文件命名规则

**所有 `{SENTINEL_NAME}` 生成的文件，文件名必须包含 `[{SENTINEL_NAME}]` 标记。**

- 分析结果：`{WORKSPACE_DIR}/YYYYMMDD_[{SENTINEL_NAME}]_<描述>.md`
- 金句存档：`{WORKSPACE_DIR}/YYYYMMDD_[{SENTINEL_NAME}]_金句.md`
- 状态文件：`{WORKSPACE_DIR}/sentinel-status.md`

**目录不存在时自动创建**（写入文件前先 `mkdir -p <目录>`）。

目的：用户让 AI 搜索 `[{SENTINEL_NAME}]` 即可一键定位所有相关文档。

## 目标声明

**所有新妙记录音在扫描周期内被处理。有价值内容已提取并存档到本地。**

每次扫描开始时声明此目标，扫描结束前必须通过自检：

| 检查项 | 通过条件 |
|---|---|
| last_checked 已更新 | 当前时间与 last_checked 差距合理 |
| 新录音全部处理 | 所有本次发现的 token 都在已处理列表中 |
| 失败队列可控 | 无新增失败项，或失败项已记录原因 |
| 结果已存档 | 有内容 → 文件已写入；无内容 → 明确标记"无" |

未通过项必须记录原因并加入失败队列，不能静默跳过。

## 扫描规则

### 搜索范围
- 起始时间：`last_checked` 时间
- 结束时间：当前时间 + 1 天 buffer
- 命令：`lark-cli minutes +search --start <last_checked日期> --end <今天日期+1> --as user`
- **必须分页遍历**：使用 --page-token 获取所有页，不能只读第一页

### 排除规则
1. 已在"已处理 token"列表中的 → 跳过
2. 已清除标题的录音 → 跳过

---

## 分类规则（简化版）

只分两类：

| 类别 | 判断条件 | 处理方式 |
|---|---|---|
| 短录音 | 时长 < 5 分钟 | 读逐字稿 → 提取待办/素材/金句 → 本地存档 |
| 长录音 | 时长 ≥ 5 分钟 | 读逐字稿 → 提取核心摘要 + 待办/素材/金句 → 本地存档 |

**注意**：标题含 "soundcore Work" **不是**低价值判断依据。这是耳机自动录音的默认标题，可能包含高价值内容，必须按时长分类处理。

## 提取规则

从**逐字稿**中提取信息（智能纪要只作参考）：

### 1. 待办/行动项
- 识别明确的 action items
- 标注优先级（高/中/低）和 deadline（如有）

### 2. 核心摘要（长录音专用）
- 一句话概括录音主题
- 提取 3-5 个关键要点
- 识别具体可执行的方案/路径

### 3. 内容素材
- **金句**：可发朋友圈/短视频文案，标注 [来源:原话]
- **观点/案例**：可写成公众号/社媒帖子

### 4. 金句存档
- 候选金句 → `{WORKSPACE_DIR}/YYYYMMDD_[{SENTINEL_NAME}]_金句.md`
- 格式：`> "原话" — 上下文说明`
- **必须从逐字稿（说话人原话）提取**

## 获取转录的正确路径

⚠️ **重要：优先读逐字稿，智能纪要只作辅助参考**。智能纪要是 AI 二次总结，丢失大量原话细节。

**主路径（逐字稿优先，必须执行）：**

```
lark-cli vc +notes --minute-tokens "<token1>,<token2>,..." --as user --format pretty --overwrite
   → 下载逐字稿到 minutes/<token>/transcript.txt
   → 读取 transcript.txt 提取内容
```

**辅助路径（智能纪要，用于快速了解结构）：**

```
1. lark-cli docs +search --query "<标题关键词>" --as user --format pretty
   → 搜索智能纪要文档，获取 doc_token

2. lark-cli docs +fetch --doc <doc_token> --as user --format pretty
   → 读取智能纪要
```

**备选路径（逐字稿下载失败时）：**

```
1. lark-cli docs +search --query "<标题关键词>" --as user --format pretty
   → 搜索文字记录文档，获取 doc_token

2. lark-cli docs +fetch --doc <doc_token> --as user --format pretty
   → 读取完整转录
```

**废弃路径（不要再用）：**
- `lark-cli minutes minutes get` → 只能拿基础信息，拿不到内容
- `lark-cli minutes +get` → 不存在

**特别注意**：
- 有些录音**没有智能纪要也没有文字记录文档**，只有妙记录音 + 逐字稿
- 这种情况下必须用 `vc +notes` 获取逐字稿，否则内容完全丢失
- 禁止将录音标记为"已删除"

## 验证规则

| 步骤 | 验收标准 | 失败处理 |
|---|---|---|
| 搜索妙记 | 返回结果含 `items` 数组 | 重试 1 次，仍失败 → 记入失败队列 |
| 读转录内容 | 内容长度 > 50 字符，非错误信息 | 尝试备选路径，仍失败 → 入失败队列 |
| 结果存档 | 文件写入无报错 | 重试，仍失败 → 暂存到失败队列 |

## 状态文件维护

### 写入规则
- 每条处理完的 token 立即写入，格式：`- <token> — <标题> (<日期时间>) ✅ (<备注>)`
- 更新 `last_checked` 为当前时间
- 记录本次扫描的原始 items 数、发现的新 token 列表

### 裁剪规则
- 已处理 token 保留最近 **14 天**
- 每次扫描后检查：超过 14 天的条目 → 移到 `{WORKSPACE_DIR}/archive.md`
- 归档格式：`<原条目> — 归档于 <日期>`

### 失败队列
- 失败条目保留在队列中，每次扫描先处理
- 成功 → 移除；仍失败 → 递增重试次数
- 最多重试 **3 次**，超过 → 标记 `abandoned` → 移到归档
- **禁止标记为"已删除"**

### 去重
- 写入前检查 token 是否已存在
- 发现重复 → 保留最早条目，删除重复的
